UNDERSTANDING YOUR DECOMPILED CODE:
===================================

What you saw in Ghidra:
-----------------------
    void entry(void)
    {
        __security_init_cookie();  ‚Üê Security feature (buffer protection)
        FUN_00644de();             ‚Üê THE REAL PROGRAM STARTS HERE!
        return;
    }

What each part means:
---------------------
    __security_init_cookie() = Stack protection initialization
    FUN_00644de()            = Main program logic (analyze this next!)


GHIDRA KEYBOARD SHORTCUTS:
==========================
    G              = Go to address
    L              = Rename function/variable
    ;              = Add comment
    Ctrl+E         = Edit function signature
    Ctrl+Shift+F   = Search text
    Ctrl+D         = Bookmark


WHAT TO LOOK FOR IN FUN_00644de:
=================================

GUI Application:
    - RegisterClassEx, CreateWindowEx
    - Message loop (GetMessage, DispatchMessage)

Background Service:
    - StartServiceCtrlDispatcher
    - Service control handlers

Process Launcher:
    - CreateProcess, ShellExecute
    - Paths to other .exe files

Network Activity:
    - WSAStartup, socket, connect
    - IP addresses in strings


QUICK SEARCH COMMANDS:
======================

Find network strings:
    findstr /i "http https socket connect server ip" BGStart\strings.txt

Find file paths:
    findstr /i "C:\ appdata program" BGStart\strings.txt

Find registry keys:
    findstr /i "HKEY registry software" BGStart\strings.txt
